## Directory Purposes

**AHK/ — Frontend GUI Application:**
- Purpose: Windows user interface for radiologists to input draft findings and review formatted reports
- Contains: AutoHotkey v2 scripts, DLL binaries, runtime logs
- Key files: `main_dll.ahk` (entry), `include/*.ahk` (modules)
- Built with: AutoHotkey v2.0 interpreter
- Runtime output: `ggwave_log.txt` (session log)

**python-backend/ — Report Processing Pipeline:**
- Purpose: Receive draft from frontend over audio, process through LLM pipeline, return formatted report
- Contains: Python modules, report templates (planned), runtime logs
- Key files: `backend.py` (entry), `lib/*.py` (modules)
- Built with: Python 3 with ggwave, pyaudio
- Runtime output: `backend_log.txt` (session log), rendered reports (in memory)

**ggwave-wrapper/ — C/C++ DLL Wrapper:**
- Purpose: Simplified C interface to ggwave library for AHK/Windows integration
- Contains: C++ source, CMake config, DLL binary
- Key files: `ggwave_simple.h` (API), `ggwave_simple.cpp` (impl)
- Built with: CMake + MSVC or mingw-w64
- Output: `build/ggwave_simple.dll` (used by AHK frontend)

**ggwave/ — Upstream ggwave Library:**
- Purpose: Data-over-sound encoding/decoding (third-party git submodule)
- Contains: C/C++ source for audio codec
- Not modified in this project; referenced by ggwave-wrapper build

**build/ — CMake Build Output:**
- Purpose: Intermediate and final build artifacts
- Generated by: CMake during build of ggwave-wrapper
- Contains: Object files, DLLs, test binaries
- Status: Generated directory, not committed

## Key File Locations

**Entry Points:**
- `AHK/main_dll.ahk`: AHK frontend application start
- `python-backend/backend.py`: Python backend application start

**Configuration:**
- `AHK/include/config.ahk`: AHK globals (compression threshold, chunk sizes, timeouts, logging)
- `python-backend/lib/config.py`: Python constants (matching AHK config values)
- `CLAUDE.md`: Project overview and architecture
- `AHK/CLAUDE.md`: Frontend-specific details
- `python-backend/CLAUDE.md`: Backend-specific details

**Core Message Handling:**
- `AHK/include/chunking.ahk`: Outbound: ChunkMessage(); Inbound: reassembly buffer, ProcessAudio()
- `python-backend/lib/chunking.py`: Outbound: chunk_message(); Inbound: handle_received_chunk()

**Compression:**
- `AHK/include/compression.ahk`: CompressString() (ntdll API), DecompressString()
- `python-backend/lib/compression.py`: lznt1_compress(), lznt1_decompress()

**Pipeline Processing:**
- `python-backend/lib/pipeline.py`: ReportPipeline (abstract), TestPipeline (echo), LLMPipeline (stub)

**Logging:**
- `AHK/include/logging.ahk`: InitializeLog(), LogMessage()
- `python-backend/lib/config.py`: setup_logging()
- Output: `AHK/ggwave_log.txt`, `python-backend/backend_log.txt`

**Audio/DLL Interface:**
- `AHK/include/dll_manager.ahk`: LoadGGWaveDll(), UnloadGGWaveDll()
- `AHK/include/gui.ahk`: SelectAudioDevices() (enumerates devices via DLL)
- `ggwave-wrapper/ggwave_simple.h`: C API definition

## Naming Conventions

**Files:**
- AHK scripts: `.ahk` extension, lowercase with underscores (e.g., `compression.ahk`, `dll_manager.ahk`)
- Python modules: `.py` extension, lowercase with underscores (e.g., `compression.py`)
- Headers: `.h` for C headers (e.g., `ggwave_simple.h`)
- Implementation: `.cpp` for C++ source (e.g., `ggwave_simple.cpp`)
- Logs: `*_log.txt` (e.g., `ggwave_log.txt`, `backend_log.txt`)
- Markdown: `.md` extension (e.g., `CLAUDE.md`, `README.md`)

**Directories:**
- Feature grouping: lowercase, plural or descriptive (e.g., `include/`, `lib/`, `rpt_templates/`)
- Build output: `build/` convention
- Hidden metadata: `.*` dot-prefix (e.g., `.git/`, `.vscode/`, `.planning/`)

**Functions/Procedures (AHK):**
- PascalCase: `GenerateMessageID()`, `ChunkMessage()`, `DecompressString()`
- Handler callbacks: `OnOK()`, `OnCancel()`, `ProcessAudio()`

**Functions/Classes (Python):**
- snake_case: `chunk_message()`, `lznt1_compress()`, `handle_received_chunk()`
- Classes: PascalCase: `ReportPipeline`, `TestPipeline`, `LLMPipeline`

**Global Variables (AHK):**
- ALL_CAPS_SNAKE_CASE: `COMPRESSION_THRESHOLD`, `GGWAVE_PAYLOAD_LIMIT`, `LOG_ENABLED`
- State variables: lowercase: `selectedSpeakerIndex`, `messageCounter`, `chunkReceiveBuffer`

**Constants (Python):**
- ALL_CAPS_SNAKE_CASE: `COMPRESSION_THRESHOLD`, `GGWAVE_PAYLOAD_LIMIT`, `LOG_ENABLED`

**Message Fields:**
- Short abbreviations (2 chars): `id` (message ID), `fn` (function), `ct` (content), `st` (status), `ci` (chunk index), `cc` (chunk count), `z` (compression flag)
- Rationale: Minimize JSON payload size for bandwidth constraints

## Where to Add New Code

**New Feature (e.g., new LLM stage):**
- Primary code: `python-backend/lib/pipeline.py` → add method to `LLMPipeline` class
- Backend tests: Create new test script in `python-backend/test_*.py` (if testing independently)
- Documentation: Update `python-backend/CLAUDE.md` with stage details
- AHK response handling: May need updates in `AHK/main_dll.ahk` or `AHK/include/chunking.ahk` for new response fields

**New Report Template:**
- Implementation: Create markdown file in `python-backend/rpt_templates/` following template format (study name, aliases, field list, skeleton)
- No code changes needed: Template index built at startup via frontmatter scanning (planned in LLMPipeline.__init__)

**New Utility Function (shared across modules):**
- AHK: Place in appropriate existing module (e.g., compression.ahk) or create new `include/*.ahk`
- Python: Add to relevant module in `python-backend/lib/` or create new module
- Export in: `python-backend/lib/__init__.py` (Python only)

**GUI Improvements:**
- All GUI logic: `AHK/include/gui.ahk`
- Entry point: Modify `AHK/main_dll.ahk` Main() to call new GUI functions
- Test: Run modified script with AutoHotkey v2 interpreter

**Backend Testing:**
- Unit/integration tests: Create `python-backend/test_<feature>.py`
- Run: `python test_<feature>.py` (scripts must import from `lib`)
- Audio testing: See `python-backend/test-output-audio.py`, `python-backend/testing-audio.py` (reference implementations, not part of main flow)

**Transport/Compression Changes (Critical):**
- Both sides must be updated together or pipe breaks
- AHK changes: `AHK/include/chunking.ahk` or `AHK/include/compression.ahk`
- Python changes: `python-backend/lib/chunking.py` or `python-backend/lib/compression.py`
- Configuration: Keep `AHK/include/config.ahk` and `python-backend/lib/config.py` in sync
- Testing: Use loopback audio or USB cable between test machines to verify round-trip

## Special Directories

**build/ — CMake Output:**
- Purpose: Intermediate build artifacts and final DLL
- Generated: By CMake during ggwave-wrapper build
- Committed: No (listed in .gitignore)
- Cleanup: `rm -rf build/` before rebuild

**rpt_templates/ — Report Templates:**
- Purpose: Markdown files defining study types and report structures
- Generated: No (manually authored)
- Committed: Yes (these are configuration)
- Format: Markdown with YAML-like frontmatter (syntax TBD)

**ggwave-wrapper/build/ — Wrapper Build Output:**
- Purpose: Intermediate artifacts during ggwave wrapper compilation
- Generated: By CMake
- Committed: No
- Output DLL: `ggwave_simple.dll` (copied to AHK/ after build)

**Documentation Directories:**
- `documentation/`: Narrative design docs (e.g., minimodem migration plan)
- `docs/`: Architecture diagrams or reference (if any)
- `.planning/codebase/`: Generated analysis documents (this directory)
