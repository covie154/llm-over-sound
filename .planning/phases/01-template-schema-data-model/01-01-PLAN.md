**Module docstring:** "Pydantic models for radiology report template validation and LLM output schemas."

**Imports:**
```python
from __future__ import annotations
import re
import keyword
from typing import Optional
from pydantic import BaseModel, ConfigDict, Field, create_model, field_validator, model_validator
```

**Constants:**
- `NOT_DOCUMENTED = "__NOT_DOCUMENTED__"` -- sentinel for unreported fields (per D-29)
- `PLACEHOLDER_PATTERN = re.compile(r"\{\{(\w+(?::\w+)?)\}\}")` -- regex matching `{{field_name}}` and `{{type:name}}` placeholders (per D-01 through D-04)
- `FIELD_NAME_PATTERN = re.compile(r"^[a-z][a-z0-9_]*$")` -- lowercase snake_case field names starting with letter (per Pitfall 2 in research)

**Class: FieldDefinition(BaseModel)** -- per D-11, D-12, TMPL-02, TMPL-03
- `model_config = ConfigDict(extra='forbid')` (per D-26)
- `name: str` -- field name, must match FIELD_NAME_PATTERN
- `normal: str` -- default normal text authored by radiologist
- `sex: Optional[str] = None` -- `"male"`, `"female"`, or `None` for universal fields (per D-12)
- `@field_validator('name')`: Validate `name` matches FIELD_NAME_PATTERN and is not a Python keyword (`keyword.iskeyword(v)`). Error message: `f"Field name '{v}' must be lowercase snake_case starting with a letter and not a Python keyword"`
- `@field_validator('sex')`: Validate `sex` is one of `None`, `"male"`, `"female"`. Error message: `f"sex must be 'male', 'female', or null, got '{v}'"`

**Class: GroupPartial(BaseModel)** -- per D-08
- `model_config = ConfigDict(extra='forbid')`
- `members: list[str]` -- subset of group members that are normal
- `text: str` -- combined normal text for this subset

**Class: FieldGroup(BaseModel)** -- per D-05, D-06, D-07, TMPL-07
- `model_config = ConfigDict(extra='forbid')`
- `name: str` -- group name
- `members: list[str]` -- list of field names in this group (min 2)
- `joint_normal: str` -- combined normal text when all members are normal
- `partials: list[GroupPartial] = []` -- optional pre-authored partial combinations (per D-08, D-10)
- `@field_validator('members')`: Validate `len(members) >= 2`. Error message: `f"Group must have at least 2 members, got {len(v)}"`
- `@model_validator(mode='after')`: Validate each partial's members are a subset of group members. Error: `f"Partial members {set(partial.members) - set(self.members)} not in group '{self.name}' members"`

**Class: TemplateSchema(BaseModel)** -- per FWRK-03, TMPL-01, TMPL-08, TMPL-09
- `model_config = ConfigDict(extra='forbid')` (per D-26)
- `study_name: str`
- `aliases: list[str]` -- at least one alias required
- `fields: list[FieldDefinition]` -- ordered field list (YAML order is canonical per D-11)
- `groups: list[FieldGroup] = []`
- `technique: str` -- boilerplate technique text (per TMPL-08)
- `interpolate_normal: bool = False` -- default OFF per CLAUDE.md constraint
- `impression: bool = True` -- default ON per TMPL-05
- `important_first: bool = False` -- default OFF per TMPL-06
- `@field_validator('aliases')`: Validate `len(aliases) >= 1`. Error: `"At least one alias is required"`
- `@field_validator('fields')`: Validate `len(fields) >= 1`. Error: `"At least one field is required"`
- `@model_validator(mode='after')`: Cross-validate groups against fields list:
  1. Build `field_names = {f.name for f in self.fields}`
  2. Check each group member exists in field_names. Error: `f"Group '{group.name}' references field '{member}' which is not in the fields list. Available fields: {sorted(field_names)}"`  (per D-06)
  3. Track `seen_in_groups: set[str]` -- if a field appears in multiple groups, error: `f"Field '{member}' appears in multiple groups. Each field can belong to at most one group."` (per D-07)
  4. Check for duplicate field names in the fields list. Error: `f"Duplicate field name: '{name}'"`

**Class: StudyTypeClassification(BaseModel)** -- per D-21
- `model_config = ConfigDict(extra='forbid')`
- `study_type: str` -- the classified study type (constrained to known aliases at runtime)
- `confidence: float = Field(ge=0.0, le=1.0)` -- confidence score 0.0-1.0

**Function: create_findings_model(schema: TemplateSchema, technique_fields: list[str] | None = None) -> type[BaseModel]** -- per FWRK-04, D-16, D-17, D-20
- Docstring: "Generate a dynamic Pydantic model from a template's field list. Each template field becomes an Optional[str] field defaulting to None. None means unreported (maps to __NOT_DOCUMENTED__ or normal text depending on interpolate_normal). Technique fields are included alongside anatomical fields per D-20."
- Build `field_definitions: dict` where each `field_def.name` from `schema.fields` maps to `(Optional[str], None)`
- If `technique_fields` is provided, add each technique field name as `(Optional[str], None)`
- Generate model name: `f"{schema.study_name.replace(' ', '')}Findings"`
- Call `create_model(model_name, **field_definitions)` with `__config__=ConfigDict(extra='forbid')`
- Return the model class

**Function: validate_body_placeholders(schema: TemplateSchema, body: str) -> list[str]** -- per D-28
- Docstring: "Cross-validate frontmatter fields against body placeholders. Returns a list of warning/error strings. Empty list means no issues."
- Extract all placeholders from body using PLACEHOLDER_PATTERN
- Separate into plain field placeholders (no colon prefix) and typed placeholders (measurement:, technique:)
- Build `schema_field_names = {f.name for f in schema.fields}`
- Check 1: Fields in schema but not referenced in body as `{{field_name}}`. Warning: `f"Field '{name}' defined in frontmatter but has no placeholder in template body"`
- Check 2: Plain placeholders in body not in schema field names. Error: `f"Placeholder '{{{{name}}}}' in template body has no matching field in frontmatter"`
- Return the list of warning/error strings
