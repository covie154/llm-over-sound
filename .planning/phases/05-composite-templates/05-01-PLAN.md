    Create `python-backend/lib/templates/composer.py` with the following functions:

    **compose_template(composite: LoadedTemplate, bases: dict[str, LoadedTemplate], resolution_chain: set[str] | None = None) -> LoadedTemplate**

    Main entry point. Takes a raw-parsed composite LoadedTemplate (parsed from frontmatter only, body placeholder validation skipped), a dict of already-loaded base templates keyed by relative path, and an optional resolution_chain for circular detection.

    Implementation:
    1. Circular detection: If `composite.file_path` is in `resolution_chain`, raise `TemplateLoadError(composite.file_path, "Circular composition detected: ...")`. Initialize resolution_chain as empty set if None, add composite.file_path.
    2. Resolve base templates: For each path in `composite.schema.composable_from`, look it up in `bases` dict. If missing, raise `TemplateLoadError(composite.file_path, f"Base template not found: {ref_path}")` per D-27.
    3. Validate exclusions: Call `_validate_exclusions(composite.schema.exclude_fields, resolved_bases, composite.file_path)` -- checks that every excluded field name actually exists in its base (D-28), and that exclude_fields keys are all in composable_from.
    4. Merge fields: Call `_merge_fields(resolved_bases, composite.schema.exclude_fields, composite.schema.fields)` -- returns ordered list of FieldDefinition.
    5. Merge groups: Call `_merge_groups(resolved_bases, composite.schema.exclude_fields, merged_field_names, composite.schema.groups)`.
    6. Build composed TemplateSchema with: study_name, aliases, technique, interpolate_normal, impression, important_first, variant all from composite.schema (per D-03, COMP-03). Fields and groups from merge results. Set composable_from=None and exclude_fields=None on the composed schema (it's now a "resolved" template).
    7. Run `validate_body_placeholders(composed_schema, composite.body)` -- if issues, raise TemplateValidationError per Pitfall 5.
    8. Return `LoadedTemplate(schema=composed_schema, body=composite.body, file_path=composite.file_path)`.

    **_validate_exclusions(exclude_fields: dict[str, list[str]] | None, resolved_bases: list[tuple[str, LoadedTemplate]], composite_path: str) -> None**
    - If exclude_fields is None, return immediately.
    - For each key in exclude_fields, verify it's a path in resolved_bases. If not, raise TemplateLoadError.
    - For each (path, field_names) in exclude_fields, get the base's field names set. For each excluded name, if not in the base's fields, raise `TemplateLoadError(composite_path, f"Excluded field '{name}' does not exist in base template '{ref_path}'")`.

    **_merge_fields(resolved_bases: list[tuple[str, LoadedTemplate]], exclude_fields: dict[str, list[str]] | None, composite_fields: list[FieldDefinition]) -> list[FieldDefinition]**
    - Build exclusion set per base from exclude_fields (or empty dict if None).
    - Iterate resolved_bases in order. For each base, iterate its fields. Skip if field.name is in that base's exclusion set. If field.name already in seen_names set, raise `TemplateLoadError` (collision per D-25).
    - Append composite's own fields at end. Check for collisions with seen_names.
    - Return merged list.

    **_merge_groups(resolved_bases: list[tuple[str, LoadedTemplate]], exclude_fields: dict[str, list[str]] | None, merged_field_names: set[str], composite_groups: list[FieldGroup]) -> list[FieldGroup]**
    - Build exclusion set per base.
    - For each base, iterate its groups. Per D-11: if ANY member of the group is in that base's exclusion set, skip the entire group. Also verify all remaining members exist in merged_field_names.
    - Append composite's own groups at end.
    - Return merged list.
