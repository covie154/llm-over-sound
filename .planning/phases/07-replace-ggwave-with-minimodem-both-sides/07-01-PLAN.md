<objective>
Build the foundation of `minimodem-wrapper/`: scaffold the CMake project mirroring `ggwave-wrapper/` / `quiet-wrapper/`, vendor the minimodem core DSP sources, refactor `minimodem.c`'s `main()` TX/RX loops into reusable context-struct-based library functions, implement the new WinMM audio backend, and produce a dependency-free `minimodem_simple.dll`.

This is the highest-correctness-risk work in the phase. It deliberately stops short of the public API + RX thread + loopback gate (Plan 02) so that the `main()` decomposition and the WinMM backend each get full context budget.

Purpose: Per CONTEXT.md locked decisions, both sides move to minimodem FSK. The DLL must mirror `ggwave_simple.h` and ship as a single self-contained binary (WinMM, static fftw3f) so AHK loads it unchanged. The Wave 0 gate (loopback + CRC vector, Plan 02) cannot run until this compiles.
Output: A compiling `minimodem_simple.dll` containing the decomposed FSK core and the WinMM backend, verified dependency-free with `ntldd`.
</objective>
