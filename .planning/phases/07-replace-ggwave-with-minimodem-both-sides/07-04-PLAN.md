<objective>
Repoint the Python backend transport from ggwave to minimodem: add a `lib/minimodem.py` ctypes binding to `libminimodem_simple.so` (symmetric API), add `crc32_str` (matching the proven vector), convert chunking to single-frame + `crc` with full retransmit, swap transport init in `backend.py` to `--baud` (default 1200) and remove `--protocol`, and drop the `ggwave`/`pyaudio` transport imports. The 5-stage LLM pipeline is untouched.

Purpose: Per CONTEXT.md, the Pi compiles the same wrapper as `libminimodem_simple.so` (ALSA/Pulse) and calls it via ctypes — symmetric with AHK. CLAUDE.md: any framing/encoding change MUST be mirrored — this mirrors Plan 03's AHK single-frame + crc exactly so the pipe interoperates. Runtime State Inventory: an external Pi service unit invoking `backend.py -p/--protocol` will break when the flag is removed — call this out for the user.
Output: A Python backend that receives/sends minimodem FSK single frames with CRC32 integrity, configurable by `--baud`.
</objective>
