  <action>
    Implement all 12 exports declared in `minimodem_simple.h`. Hold one module-level `minimodem_ctx`, one `pthread_mutex_t` (prefer pthreads — portable to the Linux build per RESEARCH Threading §4), an `is_transmitting` atomic, a decoded-byte ring/buffer + a newline accumulator, and a `char error[256]`.

    - `minimodem_simple_init(int playbackDeviceId, int captureDeviceId, int baud)`: call `mm_build_config(&ctx, baud, 48000)` (return its negative error if it fails); open the RECORD stream (`ctx.sa_in`) and PLAYBACK stream (`ctx.sa_out`) via `simpleaudio_open_stream` with the WinMM backend on Windows / ALSA|Pulse on Linux, 48k mono, RX float (TX float is fine too). Pass the device IDs through as the backend's integer index. Then create the background RX thread.
    - Background RX thread: loop `mm_rx_step(&ctx, tmp, sizeof tmp)`; lock; append decoded bytes to the queue; when a `\n` is seen, mark a complete line available; unlock. Per Pitfall 3 (half-duplex): if `is_transmitting` is set, still call `mm_rx_step` to keep the buffer drained but DISCARD the bytes (read-and-discard) — the safe default; on a clean wired link this is harmless. Per Security V5: cap accumulated line length (e.g. 8192) and the queue size; drop/reset on overflow rather than growing unbounded.
    - `minimodem_simple_send(const char* message, int volume)`: set `is_transmitting=1`; map `volume/100.0`→`tx_amplitude` and `simpleaudio_tone_init(4096, amplitude)` (Open Question 4); call `mm_tx_bytes(&ctx, (const unsigned char*)message, strlen(message))`; when the waveOut drain completes set `is_transmitting=0`; return 0 / negative on error.
    - `minimodem_simple_is_transmitting()`: return the atomic flag (AHK busy-waits on it).
    - `minimodem_simple_process()`: near no-op returning 0 — the RX thread does the pumping (the old ggwave `_process` pumped SDL; preserve the signature so AHK's 10 ms poll is unchanged). Optionally do a light timeout sweep, never blocking work (Open Question 3).
    - `minimodem_simple_receive(char* buffer, int bufferSize)`: lock; if a complete newline-delimited line is queued, copy up to `bufferSize` bytes (BOUNDS-CHECKED, never overrun — Security V5) WITHOUT the trailing `\n`, remove it from the queue, return its length; else return 0; unlock. Mirror the `ggwave_simple_receive` contract (length, 0 if none, negative on error).
    - `minimodem_simple_set_baud(int baud)`: rebuild config/fsk_plan for the new baud (replaces `set_protocol`); both ends must match — this changes only this end.
    - `minimodem_simple_cleanup()`: signal the RX thread to stop, `pthread_join`, `mm_destroy(&ctx)`, destroy mutex.
    - device count/name getters: delegate to the WinMM `mm_*_device_count`/`mm_*_device_name` helpers (Windows) or the simpleaudio enumeration (Linux).
    - `minimodem_simple_get_error()`: return `ctx.error` / module error buffer.

    Use one mutex around ALL queue access on both producer (RX thread) and consumer (`_receive`) per Pitfall 7.
  </action>
